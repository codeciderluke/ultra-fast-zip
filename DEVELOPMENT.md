# Ultra Fast Zip — Development History

> Developed: 2026-07-09 · Version 1.0 · Designed by Codecider Lab
>
> Renamed on 2026-07-14 from **FastPack Studio (.fpk)** to **Ultra Fast Zip (.ufz)** (§2.8).
> Fully internationalized to English on the same date (§2.10).

## 1. Overview

A Windows-first GUI + CLI tool for rapidly packing/unpacking folders with many
small files, using its own `.ufz` format (block-based Zstandard, xxHash64).

- **GUI**: PySide6, dark theme, 4 tabs (Compress / Extract / Inspect / Settings)
- **CLI**: `ufz pack / unpack / inspect` — no PySide6 required
- **Python 3.11+**, packaged with PyInstaller

## 2. Development log

### 2.1 Initial implementation (spec-based MVP)

- Format v1: Header (MAGIC/VERSION/METADATA_LENGTH) + metadata JSON + zlib blocks + CRC32
- 4-tab GUI, QThread + worker + signal architecture, progress bar + color log console
- Cancellation (partial output cleanup), Zip Slip protection, CRC verification, pytest suite

### 2.2 CLI

- argparse subcommands pack/unpack/inspect, progress bar with current filename,
  `-q` flag, exit codes (0/1/130), cp949 console fallback, double-click help mode

### 2.3 Codec/checksum upgrades

| Format version | Compression | Checksum | Status |
|---|---|---|---|
| v1 | zlib | CRC32 | read-compatible |
| v2 | Zstandard | CRC32 | read-compatible |
| **v3 (current)** | **Zstandard (levels 1-22)** | **xxHash64** | written |

- Block header is `<QQQ>` (raw/comp/xxh64) from v3; v1/v2 use `<QQI>` (CRC32) — auto-detected

### 2.4 Extraction engine redesign (pipeline)

```
Archive (mmap zero-copy) -> Block Queue -> Decompress xN -> Write Queue -> Writers xN -> Disk
```

- mmap zero-copy reads, producer/consumer pipeline, 256 MB in-flight memory budget
  (semaphore backpressure; python-zstandard lacks `decompress_into`, so budget over pool)
- Directory cache (all folders created up front), tiny-file packing, sequential layout,
  FIFO dispatch; SIMD comes from the zstd/xxhash C libraries

### 2.5 Performance findings (70,208 files / 1 GB)

1. **CPython buffered `read(n)` preallocates n bytes** — chunked `read(4MB)` loops on
   tens of thousands of small files wasted 88s (80% of runtime); request only the
   remaining size
2. **pathlib in hot loops cost ~14s** — replaced with string `os.path` operations
3. **Removed per-file `exists()` stat** — exclusive-create `open(path, "xb")`

Result (warm cache, zstd-6 vs zip deflate-6): pack **11.7s** vs 34.5s (3.0x),
unpack **14.6s** vs 22.5s (1.5x), size **223.5 MB** vs 295.2 MB.

### 2.6 UX

- Current-filename status line (GUI + CLI, throttled to 20 Hz), incremental progress
- All options auto-saved via QSettings and restored on launch
- Multi-select batch jobs for both compress and extract, `[i/N]` logging

### 2.7 Packaging

- PyInstaller: GUI = onedir, CLI = onefile; icon generated by `scripts/make_icon.py`
- Batch scripts: `setup.bat` → `build.bat` → `package.bat`
  (batch files are ASCII-only: cmd parses them in the OEM codepage)

### 2.8 Rebrand + open-source preparation (2026-07-14)

- FastPack Studio → **Ultra Fast Zip**, `.fpk` → **`.ufz`**, CLI `fpk` → **`ufz`**
- New archives use magic `UFZ1\0\0\0\1`; legacy `FPK1\0\0\0\1` + `format: "FPK"`
  metadata stay readable (backward compatible, covered by tests)
- `FpkError` → `UfzError`, QSettings scope → `UltraFastZip`, log prefix `ufz_`
- MIT `LICENSE`, third-party `NOTICE.md`, `.gitignore`, git repository initialized
- Multi-format benchmark `scripts/bench_multi.py` + analysis `docs/format_comparison.md`

### 2.9 Multi-format auto-detected extraction (2026-07-14)

- New `app/core/formats.py`: magic-byte `detect_format()` + unified `extract_archive()`
  shared by the GUI worker and CLI
- Backends: zip/tar family/gz/bz2/xz/zst = stdlib (+zstandard streams);
  7z = py7zr with bsdtar fallback; rar/cab/iso = bsdtar (libarchive, Win10+/macOS built-in)
- `safe_join` validation on every path, `./name` entry normalization, cp949 zip
  filename recovery, compressed-offset progress, cancellation support
- 13 new tests; verified against real bsdtar-created zip/tar.gz/iso archives

### 2.10 Full English internationalization (2026-07-14)

- All user-facing strings (GUI, CLI, logs, errors) translated from Korean to English
- Korean-content assets (old-brand user manual PDF/HTML, GUI screenshots, original

## 4. Verification status

- pytest **33/33 passing** — header/block roundtrips, corruption detection, overwrite
  refusal, hidden-file exclusion, empty file/folder preservation, v1 (zlib) backward
  compatibility, path traversal rejection, multi-format detection/extraction, Zip Slip

## 5. Known issues / future work

- Cold-cache small-file reads can be disk/AV bound (~100 files/s observed) —
  environmental, not tool-related
- Candidates: parallel pack pipeline (packer is single-threaded; loses ~23% to
  native tar.zst), drag & drop, additional codecs (brotli/LZ4), post-pack verify
  option, recent-jobs list
